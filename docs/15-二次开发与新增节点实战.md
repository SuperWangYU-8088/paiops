# PaiOps 二次开发与新增节点实战

## 1. 开发原则

PaiOps 的扩展点很多，但新增功能前先判断应该放在哪里：

| 需求 | 首选位置 |
|---|---|
| 新的确定性运维动作或查询 | Java `NodeExecutor` |
| 队列、锁、心跳、崩溃接管 | Go Worker |
| 页面、表单、画布交互 | React 前端 |
| 长期事实和审计 | MySQL |
| 临时队列和短租约 | Redis |
| 文件与知识文档 | MinIO |
| AI 诊断和文本生成 | LLM/Agent 节点 |

不要把确定性权限判断交给 Prompt，也不要为普通业务字段创建不必要的分布式组件。

## 2. 开发环境

推荐版本：

- Java 21；
- Maven 3.9+；
- Node 22；
- npm 10+；
- Go 1.24；
- Docker Engine 24+；
- Docker Compose v2。

只使用 Docker 也能构建生产镜像，但本地安装语言工具链能更快运行单元测试和格式化。

## 3. 新增节点的完整链路

一个真正可用的新节点至少涉及：

1. 唯一 `nodeType`；
2. 后端节点定义；
3. 后端执行器；
4. 风险等级；
5. 前端图标、颜色和摘要；
6. 前端默认值；
7. 前端配置表单；
8. 单元测试；
9. 数据库迁移兼容；
10. 使用文档和验收记录。

本文用只读节点 `latency_threshold` 演示：输入响应耗时，判断是否超过阈值。

## 4. 第一步：定义节点协议

先写清楚输入、配置和输出，不要先写 UI。

```json
{
  "nodeType": "latency_threshold",
  "input": {
    "latencyMs": 850
  },
  "config": {
    "thresholdMs": 1000
  },
  "output": {
    "healthy": true,
    "latencyMs": 850,
    "thresholdMs": 1000,
    "exceededByMs": 0
  }
}
```

约定：

- `latencyMs` 必须是非负数；
- `thresholdMs` 范围 1～600000；
- 等于阈值视为健康；
- 节点不访问网络、不修改外部系统，因此是只读风险。

## 5. 第二步：实现 Java 执行器

文件建议：

```text
backend/src/main/java/com/paiagent/engine/executor/impl/LatencyThresholdNodeExecutor.java
```

示例：

```java
package com.paiagent.engine.executor.impl;

import com.paiagent.engine.executor.NodeExecutor;
import com.paiagent.engine.model.WorkflowNode;
import org.springframework.stereotype.Component;

import java.util.LinkedHashMap;
import java.util.Map;

/**
 * 响应耗时阈值节点。
 *
 * 该节点只做确定性数值判断，不访问外部系统，因此可以安全用于自动诊断流程。
 */
@Component
public class LatencyThresholdNodeExecutor implements NodeExecutor {

    @Override
    public Map<String, Object> execute(WorkflowNode node, Map<String, Object> input) {
        double latencyMs = requiredNonNegativeNumber(input.get("latencyMs"), "缺少 latencyMs");
        double thresholdMs = boundedNumber(node.getData().get("thresholdMs"), 1000, 1, 600_000);

        Map<String, Object> output = new LinkedHashMap<>();
        output.put("healthy", latencyMs <= thresholdMs);
        output.put("latencyMs", latencyMs);
        output.put("thresholdMs", thresholdMs);
        output.put("exceededByMs", Math.max(0, latencyMs - thresholdMs));
        return output;
    }

    @Override
    public String getSupportedNodeType() {
        return "latency_threshold";
    }

    private double requiredNonNegativeNumber(Object value, String message) {
        if (!(value instanceof Number number) || number.doubleValue() < 0) {
            throw new IllegalArgumentException(message);
        }
        return number.doubleValue();
    }

    private double boundedNumber(Object value, double fallback, double min, double max) {
        double parsed = value instanceof Number number ? number.doubleValue() : fallback;
        if (parsed < min || parsed > max) {
            throw new IllegalArgumentException("thresholdMs 必须在 1 到 600000 之间");
        }
        return parsed;
    }
}
```

`NodeExecutorFactory` 会自动收集 Spring Bean，不需要新增 `switch` 或手工注册。

## 6. 第三步：注册节点定义

编辑：

```text
backend/src/main/java/com/paiagent/service/NodeDefinitionService.java
```

在 `addOpsNodeDefinitions` 中增加：

```java
definitions.putIfAbsent("latency_threshold", createNodeDefinition(
        "latency_threshold",
        "响应耗时阈值",
        "OBSERVABILITY",
        "LT",
        "{\"type\":\"object\",\"required\":[\"latencyMs\"],\"properties\":{\"latencyMs\":{\"type\":\"number\"}}}",
        "{\"type\":\"object\",\"properties\":{\"healthy\":{\"type\":\"boolean\"},\"latencyMs\":{\"type\":\"number\"},\"thresholdMs\":{\"type\":\"number\"},\"exceededByMs\":{\"type\":\"number\"}}}",
        "{\"type\":\"object\",\"properties\":{\"thresholdMs\":{\"type\":\"number\",\"default\":1000,\"minimum\":1,\"maximum\":600000}}}"
));
```

图标使用稳定的 ASCII 缩写 `LT`，避免依赖系统 Emoji 字体。

## 7. 第四步：声明安全等级

编辑：

```text
backend/src/main/java/com/paiagent/engine/safety/NodeSafetyPolicy.java
```

把 `latency_threshold` 加入只读集合。不要让未知节点自动继承低风险等级；未知类型应该保守拒绝或要求治理。

风险分类建议：

| 等级 | 示例 | 自动执行 |
|---|---|---|
| `READ_ONLY` | 查询、解析、阈值判断 | 可按策略自动 |
| `LOW_RISK` | 通知、受控无状态重启 | 人工启动或策略控制 |
| `HIGH_RISK` | 扩缩容、回滚、数据库写入 | 必须审批 |
| `GOVERNANCE` | 门禁、人工审批 | 流程控制 |

## 8. 第五步：补后端单元测试

文件：

```text
backend/src/test/java/com/paiagent/engine/executor/impl/LatencyThresholdNodeExecutorTest.java
```

示例：

```java
package com.paiagent.engine.executor.impl;

import com.paiagent.engine.model.WorkflowNode;
import org.junit.jupiter.api.Test;

import java.util.Map;

import static org.junit.jupiter.api.Assertions.*;

class LatencyThresholdNodeExecutorTest {

    private final LatencyThresholdNodeExecutor executor = new LatencyThresholdNodeExecutor();

    @Test
    void shouldPassWhenLatencyEqualsThreshold() {
        WorkflowNode node = new WorkflowNode();
        node.setData(Map.of("thresholdMs", 1000));

        Map<String, Object> output = executor.execute(node, Map.of("latencyMs", 1000));

        assertEquals(Boolean.TRUE, output.get("healthy"));
        assertEquals(0.0, output.get("exceededByMs"));
    }

    @Test
    void shouldReportExceededAmount() {
        WorkflowNode node = new WorkflowNode();
        node.setData(Map.of("thresholdMs", 800));

        Map<String, Object> output = executor.execute(node, Map.of("latencyMs", 950));

        assertEquals(Boolean.FALSE, output.get("healthy"));
        assertEquals(150.0, output.get("exceededByMs"));
    }

    @Test
    void shouldRejectNegativeLatency() {
        WorkflowNode node = new WorkflowNode();
        node.setData(Map.of("thresholdMs", 800));

        assertThrows(
                IllegalArgumentException.class,
                () -> executor.execute(node, Map.of("latencyMs", -1))
        );
    }
}
```

运行：

```bash
JAVA_HOME=/usr/lib/jvm/java-21-openjdk-amd64 \
  mvn -f backend/pom.xml -Dtest=LatencyThresholdNodeExecutorTest test
```

再跑全量：

```bash
JAVA_HOME=/usr/lib/jvm/java-21-openjdk-amd64 mvn -f backend/pom.xml test
```

## 9. 第六步：前端节点外观

至少检查三个文件：

```text
frontend/src/utils/workflowNode.ts
frontend/src/components/FlowCanvas.tsx
frontend/src/pages/EditorPage.tsx
```

### 9.1 默认节点数据

在 `workflowNode.ts` 的节点类型分支中为 `thresholdMs` 设置默认值 `1000`。默认值必须与后端一致。

### 9.2 画布样式

在 `FlowCanvas.tsx` 的外观映射中增加：

```ts
latency_threshold: {
  icon: 'LT',
  tone: 'node-tone-green',
  caption: 'Latency threshold',
},
```

固定图标尺寸，文本超长时省略，不让节点因动态内容改变宽高。

### 9.3 右侧配置表单

在 `EditorPage.tsx` 根据 `selectedNodeType === 'latency_threshold'` 渲染：

```tsx
<Form.Item label="告警阈值（毫秒）" required>
  <InputNumber
    min={1}
    max={600000}
    value={Number(selectedNode.data?.thresholdMs ?? 1000)}
    onChange={(value) => updateSelectedNodeData({ thresholdMs: value ?? 1000 })}
  />
</Form.Item>
```

后端仍必须独立校验。前端 `min/max` 只是交互约束，不是安全边界。

## 11. 第八步：数据库迁移

动态运维节点由 `NodeDefinitionService` 提供，不一定需要插入 `node_definition`。如果新功能增加表、列、索引或必须持久化的种子数据，需要同时更新：

```text
backend/src/main/resources/schema.sql
backend/src/main/java/com/paiagent/config/SchemaMigrationRunner.java
backend/src/test/resources/schema-test.sql
```

规则：

1. `schema.sql` 支持全新数据库；
2. `SchemaMigrationRunner` 支持已有数据库幂等升级；
3. 测试 schema 支持 H2，避免使用只被 MySQL 接受的重复索引名；
4. 迁移可重复执行；
5. 不删除历史列和数据，除非有明确迁移与回滚方案。

## 13. 什么时候需要改 Go Worker

普通新节点不需要改 Worker。只有以下情况才进入 `worker/`：

- 改队列协议；
- 改 Worker 并发模型；
- 改执行锁或续租；
- 改崩溃恢复；
- 改控制面内部 API 契约；
- 增加真正由 Worker 本地执行的受控任务。

改 Worker 必须补：

- 正常路径测试；
- Redis/控制面网络失败测试；
- context 取消测试；
- 锁丢失测试；
- 重复消息与幂等测试；
- 崩溃恢复测试。

运行：

```bash
cd worker
gofmt -w ./cmd ./internal
go test ./...
go vet ./...
```

不要在 Worker 中加入数据库账号。Worker 只持有 Redis 配置和内部控制面令牌。

## 14. 代码注释标准

建议使用中文解释以下内容：

- 安全边界为什么存在；
- 事务、幂等和恢复算法；
- 复杂格式或第三方协议；
- 看似可以简化但不能简化的原因；
- 回滚和兼容约束。

不要写没有信息量的注释，例如“给变量赋值”“调用方法”。代码本身能表达的内容不重复叙述。

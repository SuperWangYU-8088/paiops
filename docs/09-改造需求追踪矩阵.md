# PaiOps 改造需求追踪矩阵

## 1. 使用说明

本矩阵把《改造要求》中的方向映射到本次实际交付。状态定义：

- 已完成：已有代码、配置和验证；
- 部分完成：完成安全可用的首批场景，仍有后续扩展；
- 规划：当前版本有意不开放，需专项设计后实现。

## 9. Go 使用说明

用户要求改造部分尽量使用 Go。本次把最适合独立和高可靠实现的执行调度 Worker 作为 Go 服务，包括可靠队列、processing 列表、分布式锁、锁续租、心跳、失联接管、控制面客户端和相应测试。DAG 节点仍保留 Java，是因为现有 Spring 依赖注入、MyBatis 事务、节点工厂和快照模型都在 Java 控制面；强行跨语言拆节点会引入双重事务和大量 RPC，不利于一致性。

- 知识库导入、切分、索引、预览和检索；

- 告警、事件、Runbook、审批、动作与健康验证的状态流转。

## 4. 配置项覆盖

部署模板覆盖以下配置类别：

- 镜像与依赖源：`DOCKER_HUB_MIRROR`、`MAVEN_MIRROR_URL`、`NPM_REGISTRY`、`GOPROXY`、`ALPINE_MIRROR`；
- MySQL：数据库名、Root 密码、主机、端口与账号；
- Redis：地址、端口、密码、队列名与并发度；
- MinIO：端点、公开地址、Access Key、Secret Key 与 Bucket；
- 认证安全：JWT Secret、默认管理员账号、初始密码、Master Key；
- Worker：内部令牌、心跳间隔、接管间隔、失联阈值、锁 TTL 与续租间隔；
- 告警与出站访问：告警 Webhook Token、HTTP 允许主机名单；
- AI：Provider、Base URL、Model、API Key、温度与超时等页面配置。

真实值只能写入部署机权限为 `600` 的 `.env` 或通过系统加密配置页面保存，不得提交到 Git。

## 5. 外部联调边界

DeepSeek 已完成真实调用验证。Prometheus、Loki、Kubernetes、Alertmanager/告警平台未提供真实外部凭证，因此本基线只验证连接器实现、参数校验、安全策略和模拟测试，不声称已完成这些外部系统的生产联调。

## 6. 验收建议

部署到新环境后，应再次执行以下检查：

1. `docker compose ps` 中六个服务均为运行或健康状态；
2. 健康接口返回 `UP`；
3. 使用测试账号创建最小输入到输出 Runbook 并成功执行；
4. 从执行任务页面确认节点快照、耗时和最终输出；
5. 使用测试凭证验证审批、审计、Dry Run 和失败回滚；
6. 接入真实外部系统时先在测试命名空间和只读账号下联调。

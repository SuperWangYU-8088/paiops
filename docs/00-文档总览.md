# PaiOps 文档总览

## 1. 文档目的

本目录是 PaiOps 改造后的交付文档中心。文档按照“先理解系统、再部署、再使用、最后排障和审计”的顺序编排，适用于开发人员、平台管理员、值班运维、审批人员和审计人员。

### 2.1 初学者动手路线

如果目标是逐步理解并亲自跑通项目，建议按下面顺序：

1. 先读 `10-从零到一部署实战.md`，在测试机完成部署；
2. 再读 `11-项目架构与代码导读.md`，理解三个程序和三个基础服务；
3. 按 `12-Runbook编排与真实执行实战.md` 创建、调试并从数据库复核流程；
4. 按 `13-DeepSeek与知识库实战.md` 完成真实模型调用和 RAG；
5. 按 `14-连接器审批审计与安全实战.md` 接入测试环境；
6. 最后按 `15-二次开发与新增节点实战.md` 增加一个只读节点。

公开仓库不附带真实业务数据库。部署完成后可按 `12-Runbook编排与真实执行实战.md` 创建最小示例，并从执行任务、节点快照、Outbox 和队列四个位置核对真实执行效果。

## 3. 交付范围

本次交付包括：

- React + TypeScript 运维工作台和可视化 Runbook 编辑器；
- Java 21 / Spring Boot 控制面、确定性 DAG 引擎、事件与审批服务；
- Go Worker，包括可靠消费、执行锁续租、心跳、失联接管；
- MySQL、Redis、MinIO、后端、Worker、前端共六个容器的 Compose 部署；
- 告警 → 事件 → Runbook → 审批 → 动作 → 健康验证 → 关闭/复盘闭环；
- Prometheus、Loki、Kubernetes、HTTP、主机和数据库等查询节点；
- Kubernetes 扩缩容、滚动重启、镜像回滚三个受控高风险动作；
- 应用层 AES-GCM 凭证加密、模型密钥脱敏、内部令牌隔离、出站地址白名单；
- 中文设计、部署、运维、使用、接口、测试与追踪文档。
- 从零部署、项目导读、真实执行、DeepSeek/RAG、连接器安全和二次开发六篇实战教程。

## 4. 重要安全约定

1. `.env` 是生产密钥文件，不进入 Git，也不进入最终代码同步包。
2. `PAIOPS_MASTER_KEY` 用于解密已保存凭证，部署后必须备份，不能随意更换。
3. 告警自动执行只允许“全只读 DAG”；带写操作的 Runbook 必须人工启动并经过审批。
4. 高风险节点不接受前端传入的 `approved=true` 作为授权依据，只认数据库中属于当前执行记录、已审批且未过期的审批单。
5. SSE URL 不携带长期 JWT，只携带 60 秒有效且只能消费一次的票据。
6. 自定义 MCP 进程命令默认关闭；开启前必须完成单独安全评审。
